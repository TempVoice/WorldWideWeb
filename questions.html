QUESTIONS

Why are temporary voice channels not able to sync permissions?

This is not possible, because otherwise core features like trusted and blocked user lists of the bot would not be possible anymore. You still can set manually a role which is always trusted or blocked with the command /config role.

Option I: Recommend for a verification system.
Will be used instead of @everyone to change permissions like locking or hiding.

Option II: Recommend for a server team role.
Can always join tempoary voice channels even if they are locked, hidden or the user is blocked.

Option III: Recommend for a timeout role.
Is not able to join tempoary voice channels.

❕You can only set a single role for each option.

Why cannot the bot delete its own messages?

This is not possible, because TempVoice is build to guarantee the best possible performance. Adding such feature would result in slower processes speed and ratelimits.

Why are emojis not displayed in the bot messages?

The response message of slash command are webhooks and not a message from the bot client itself. Webhook permissions are managed by the @everyone role so you have to allow access for @everyone for "Use external Emojis".

Why do I need to wait 10 minutes to rename, claim, transfer again?

Announcement from by Mason#1337, a member of the Discord Staff:

"The new rate limit for channel NAME AND TOPIC updates is 2 updates per 10 minutes, per channel."
