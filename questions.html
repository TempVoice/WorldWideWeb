Questions?

Are temporary voice channels able to sync permissions?

This is not possible, because otherwise core features like trusted and blocked user lists of the bot would not be possible anymore.
You still can set manually a role which is always trusted or blocked with the command /config role.

- Trusted Role: Can always join tempoary voice channels even if they are locked, hidden or the user is blocked. Recommend for a server team role.
- Blocked Role: Is not able to see tempoary voice channels. Recommend for a timeout role.
- Member Role: Will be used instead of @everyone for the privacy feature. Locked mode will deny CONNECT permission and hidden will deny VIEW CHANNEL permission.

How can I change default name & limit?

Take a look at these guides: Change Default Name, Change Default Limit.

How can I integrate my verification system?

You use a member role for all verified users?
- Just set a member role with /config role.

You use a unverified member role?
- Just set a blocked role with /config role.

Why are emojis not displayed in the bot messages?

The response message of slash command are webhooks and not a message from the bot client itself. Webhook permissions are managed by the @everyone role so you have to allow access for @everyone to use EXTERNAL EMOJIS.

Why do I need to wait 10 minutes to rename, claim, transfer again?

Here is the full announcement made by Mason#1337, a member of the Discord Staff:

Hi everyone--wanted to make official note of the rate limit change on channel updates that was deployed yesterday. While most of you won't notice anything at all, there are known and unknown use cases that this effects.
The new rate limit for channel NAME AND TOPIC updates is 2 updates per 10 minutes, per channel. (Sorry that was unclear at first)

This is reflected in the normal rate limits headers so no library changes should need to happen.
For a bit of context, frequent channel updates have significant impact on our infrastructure and performance, especially in the case of large guilds and mobile clients, hence the limit changes.

Why cannot the bot delete its own messages after X seconds?  

TempVoice is build to guarantee the best possible performance. Adding such feature would result in slower processes speed.
